# Secure NVR Gateway Deployment (Edge Infrastructure)
## Overview

Проєкт присвячений розгортанню захищеного шлюзу для системи відеоспостереження в умовах складної університетської інфраструктури з подвійним NAT (Double NAT).

## Основна мета
Забезпечити стабільний зовнішній доступ для спостерігачів іспитів (ЄДКІ), ізолювавши при цьому критичну інфраструктуру від публічного доступу.

## 1. Architecture

Система побудована на базі OpenWRT, який виступає в ролі інтелектуального L3-шлюзу між загальноуніверситетською мережею та локальним сегментом відеоспостереження.

  * **Corporate Router**: Транзитний вузол з обмеженим доступом.
  * **OpenWRT Gateway**: Керування трафіком, Firewall, NTP-сервер, DHCP-сервер та NAT.
  * **Endpoints**: NVR TP-Link VIGI та камери Hikvision (різних поколінь/прошивок).

## 2. Technical Challenges & Solutions
### 2.1. Integration of Legacy & Modern Hardware (The "Zoo" Problem)

*Issue*: Різні версії прошивок камер Hikvision спричиняли помилки декодування (Error 245) у веб-інтерфейсах та мали підтримку різних версій протоколу ONVIF.

*Solution*:

  * Оновлення пристроїв до останніх доступних версії офіційних прошивок.
  * Проведено аналіз можливостей кожного пристрою та примусово встановлено кодек **H.264 (Main Profile)** замість H.265/High Profile для забезпечення максимальної сумісності з браузерами спостерігачів.

### 2.2. ONVIF Authentication & Time Synchronization

*Issue*: Після перезавантаження реєстратора камери не проходили авторизацію ONVIF через розсинхронізацію часу (Time Drift).

*Solution*:

  * OpenWRT налаштований як **локальний NTP-сервер**.
  * Впроваджено жорстку синхронізацію для камер та реєстратора (інтервал кожні 5 хвилин).

### 2.3. Security & Management Isolation (Hardening)

*Issue*:

Необхідність надати доступ до відео, але обмежити можливість зміни налаштування пристроїв.

*Solution*:

- **RBAC (Role-Based Access Control)**: Створенно окремі права доступу для зовнішнього спостерігача з обмеженням дозволів.
- **Management Plane Isolation**: Порт керування закрито для доступу з зовнішньої мережі.
- **Targeted Port Forwarding**: Порти для доступу до WebUI та потокового відео відкрито через Firewall тунелювання напряму до відеореєстратора.

## 5 🎓 Lessons Learned

Проєкт продемонстрував важливість глибокого розуміння мережевих протоколів при роботі з "Black Box" пристроями (NVR/IP-камери). Вміння діагностувати проблеми на стику L3 (Network) та L7 (Application) дозволило побудувати надійну систему в консервативному ІТ-середовищі.
