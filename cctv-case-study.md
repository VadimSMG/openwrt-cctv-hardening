# Проєкт: Налаштування системи відеоспостереження для ЄДКІ

## Стек:

  * Hikvision (IP-камери),
  * TP-Link VIGI (NVR),
  * OpenWRT (Router),
  * LMDE 7 (Workstation)
## 1. Апаратна конфігурація та підготовка

  * **Робоча станція**: Lenovo ThinkPad X61 Tablet (Middleton BIOS, SSD 64GB, 6GB RAM).
  * **ОС**: LMDE 7 (Debian 13) з середовищем XFCE.
  * **Мережевий шлюз**: Роутер з прошивкою OpenWRT.
  * **Відеореєстратор**: TP-Link VIGI NVR.
  * **Камери**: Hikvision IP-камери.

## 2. Налаштування сумісності (ONVIF & Video)

Для забезпечення стабільної передачі зображення з камер Hikvision на реєстратор VIGI було виконано:

 1. **Кодеки**: Переключено основний та додатковий потоки (Main/Sub-stream) з H.265+ для масимального терміну збереження записів на HDD (записи займають на 30-40% менше).
 2. **Активація ONVIF**: - Увімкнено протокол Integration Protocol.
 3. **Права доступу**: Створено окремого користувача ONVIF з правами Administrator.
 4. **Налаштування ONVIF**: Використано складний пароль (8+ символів, цифри, спецсимволи) для усунення помилки Parameter Error.

## 3. Мережева інфраструктура

### Статична адресація (Hard Static):
Призначено фіксовані IP-адреси безпосередньо в налаштуваннях пристроїв та в налаштуваннях шлюзу:

  * NVR: 192.168.1.10
  * Cameras: 192.168.1.21, 192.168.1.22

### NTP Синхронізація:
  * Роутер OpenWRT налаштовано як локальний NTP-сервер.
  * Всі камери та NVR синхронізовані з IP роутера для уникнення помилок авторизації ONVIF (невідповідність часових міток).

### Безпека (Firewall):
В OpenWRT створено Traffic Rules для блокування виходу камер Hikvision в інтернет (зона lan -> wan), залишивши доступ лише до локального NVR.

## 4. Керування доступом та ролями

  * **Адміністративний доступ**: Повний контроль через локальний інтерфейс та хмарний інтерфейс TP-Link CLoud.
  * **Доступ для представників ЄДКІ**:
    * Створено окремий TP-Link ID на базі університетської пошти.
    * Надано доступ через хмару з роллю Viewer (тільки перегляд та архів).
    * **Внутрішній моніторинг**: Створено локального користувача на NVR з роллю Viewer для ввімкнення/вимкнення запису без доступу до системних налаштувань.

## 5. Оптимізація запису

  * **Гігієна архіву**: Налаштовано циклічний запис (Overwrite).
  * **Режим роботи**: Використання ручного ввімкнення запису або розкладу (Schedule) саме на час проведення іспитів для економії ресурсу накопичувачів та фільтрації лекційного контенту.

### Корисні команди для діагностики (Linux)

#### Перевірка портів камери:

```Bash
    nmap -p 80,554,8000,8080 <IP_камери>
```

#### Тестування RTSP потоку:

```Bash
    vlc rtsp://user:password@<IP_камери>:554/Streaming/Channels/101
```

#### Моніторинг мережі в OpenWRT:

```Bash
    ssh root@192.168.1.1 "tcpdump -i br-lan host <IP_камери>"
```
